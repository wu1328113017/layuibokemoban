对git的理解：git是一个很好用的东西，它可以根据版本分为不同的分支，还能回退版本，也就是说我改了当前的代码如果出现了错误不知道怎么改回来可以用git回退的方式回退的之前的版本，所以代码要经常提交，最好做到一天一交或一天两交，而且每次提交都可以备注，这样就能清除哪个版本做了些什么，一般项目开发都是多人合作的形式，所以有时候会出现这样一个情况，我和另外一个同事改了同一个文件，这样就会造成冲突，但这种冲突只能自己解决，所以解决这种冲突的时候只能跟一起改过的同事商量才能解决，而有时候就算解决了git似乎也还是报冲突，所以这也是git的一个麻烦的地方，所以开发的时候尽量分工明确，不要几个人同时改一个地方，使用git有时候也会出现这样一种情况，我准备提交了，然后另外一个同事先提交了，所以我这里就报了一个冲突的错误，这是因为我提交之前的版本是另一个同事没提交之前的版本，而同事已经提交了他的一个版本，所以我的版本落后了一个版本，假如同事在原先版本的基础上对代码进行了一些调整，或是增加或是减少，不管怎么样，总之和之前的版本的代码是不一样了，而我的还是停留在之前的那个版本上，所以也会和同事的不一样，也就是和当前版本的不一样，所以这个时候git就会认为有两个以上改动了同一个文件，就会造成冲突，解决办法是要先拉取同事先提交了的版本，而git好像也会提醒你使用git pull，所以每次提交代码之前都要拉取一遍，我们称之为同步，即本地与远程仓库同步版本，说到本地和远程，这里解释以下，本地即你的电脑，你当前工作的电脑，远程即大家公用的一个服务器，本地仓库即在自己电脑上的仓库，自己一个人使用，远程仓库就是大家一起使用的仓库，提交代码到远程仓库要先将代码提交到本地仓库，在同步到远程仓库，命令在下面有写。我曾经遇到过很多git的错误，但都是冲突的小错误，所以也没怎么记得，但有一个会让我这辈子也不会忘记了，当时我是独自一人完成一个模块的开发，没有人动这个模块，仓库也没有人提交过，一直过了一个礼拜差不多，远程仓库的版本都还是没变，当时我准备提交代码了，敬遵者先拉取再提交的原则就拉取了远程仓库的代码，注意，那是一个礼拜前的版本的代码，所以一拉下来，我做的一个礼拜的心血全都付之东流，全都回到了一个礼拜前，也就是说这一个礼拜白做了，当时我犹如晴天霹雳，还好我早上有一次提交到本地仓库(还没提交到远程仓库，只是本地)的操作，使得我可以回退版本到早上，这才使得损失没那么大，所以我现在总结了这个惨痛的教训，先拉后提并没错，但在做这一步之前得将代码先提交到本地仓库，防止发生错误无法回退到安全的版本

不过要注意一下几个点：

  在github上创建的仓库如果直接在本地选择一个空的文件进行克隆(用命令的情况)，会提示你克隆了一个空的仓库，这个并无大碍，但如果你是想直接在当前目录下创建本地仓库，那么使用以下命令:

git init
git remote add origin 远程仓库分支的url
git add .
git commit -m "init"
git push -u origin master

git常用命令：

  git clone :克隆远程仓库的代码到本地

  git add . :将所有文件放入暂存区，准备提交

  git commit -m"这里写备注，说明这次提交做了些什么" :提交到本地仓库

  git push :将本地仓库推送到远程仓库中

  git pull 将远程仓库同步到工作台(即存放代码的项目中，并不是本地仓库)

  git init 初始化git环境

  git pull --rebase origin master //强制把远程库的代码跟新到当前分支上面

  git fetch //把远程仓库的代码更新到本地仓库

  git branch -r/-a //查看远程分支/全部分支

  git checkout master/branch //切换到某个分支

  •   git checkout -b test //新建test分支

  •   git checkout -d test //删除test分支
  •   git merge master //假设当前在test分支上面，把master分支上的修改同步到test分支上

  •   git merge tool //调用merge工具
  •   git stash //把未完成的修改缓存到栈容器中
  •   git stash list //查看所有的缓存
  •   git stash pop //恢复本地分支到缓存状态
  •   git blame someFile //查看某个文件的每一行的修改记录（）谁在什么时候修改的）
  •   git status //查看当前分支有哪些修改
  •   git log //查看当前分支上面的日志信息
  •   git diff //查看当前没有add的内容
  •   git diff --cache //查看已经add但是没有commit的内容
  •   git diff HEAD //上面两个内容的合并
  •   git reset --hard HEAD //撤销本地修改
  •   echo $HOME //查看git config的HOME路径

  •   export $HOME=/c/gitconfig //配置git config的HOME路径
